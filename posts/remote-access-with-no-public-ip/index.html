Remote Access/Gaming With No Public IP

Please visit Zerotier homepage here if you want to know more about it or check here for odwnloads.

In this article we will cover how to set up a Zerotier network for remote access (gaming or ssh-ing into your computer) over the internet without having an actual public IP.

In very simple words, Zerotier can be described as a tunnel that creates a local network over the internet with all the devices connected to the same network (over VPN) with every device having the same static/local IP when connected to the Zerotier network.

Creating a Zerotier Network

  1. First we will head to their homepage and get an account/log in if we already have on.

  2. After we create an account and log in, we click on the big button that says Create A Network.

  3. That will create a new network and add it to the list. Click on the network id to open the network dashboard and to set up a few things.

  4. Here are the few settings we are interested in (you are free to check out other settings as well).

    In basic settings we can change our network name, and visibility (private or public). Public means that anyone with network id can join your network. In private we have to authorize the device in order for the device to be visible in the network.

    In the ipv4 auto-assign settings we can configure how the ip of a device looks like over the Zerotier network

    And finally, the members section where we can see each member on the network and can change or add IPs for them on the network. Note: In public visibility we cannot delete members. Currently we have no devices on our network. We will add two devices in this article.

Joining a Zerotier Network (Linux)

  1. zerotier-one binary is available for pacman (Arch based distros).

    For apt and yum, check here or run:

    curl -s https://install.zerotier.com | sudo bash

  2. Then we will start the zerotier-one.service service with systemctl.

    sudo systemctl start zerotier-one.service

    and/or

    sudo sudo systemctl enable zerotier-one.service

    if we want it to start automatically on boot.

  3. Then we will copy the network id (d5e5fb6537ea1a99 in this case, replace this with your own) and run:

    sudo zerotier-cli join d5e5fb6537ea1a99

  4. If we see a 200 join OK output, everything went right. Our device will not yet show up under members on the network dashboard since we need at least 2 devices on a network to configure or see each device’s settings. Next we will use an Android device to connect to our network and see the settings for our devices.

Joining a Zerotier Network (Android)

  1. First we will head over to Play Store and download Zerotier One.

  2. Then we will click the + on top right.

  3. Then we will provide the network-id and click the checkbox that says to route all traffic through zerotier and then finally click add network.

  4. Our network should already be visible on the network list by now. We can connect to it but we cannot access the devices on the network yet because our network is private and we have not authorized the devices yet. So we will head over to our network dashboard and authorize it and also the Linux computer we added earlier by ticking the checkbox to the left of both our connected devices.

  5. Once both devices are authorized, we can toggle the switch on our Android app to connect to the network.

Joining a Zerotier Network (Windows)

For Windows, we will download the setup from here, install it and follow the exact same steps for Android, which involves entering the Network ID, enabling the network and then authorizing it in the Members section on our network dashboard.

Member Settings

Now we can tweak a few settings for every connected device. It is useful to give each device we add a name and description to identify them because there is no other way to actually identify them otherwise.

First we will go our Members section of our network dashboard.

So far(as shown in the image) we have successfully added a Linux and an Android device to our network. The names were manually added and not automatically assigned. The IPs marked with green are the ones for the respective devices. We can use them wheneer we are connected to the network.

We can verify that by a simple ping while connected to the network.

Now for ssh; we can use:

1

ssh user@192.168.195.88

from any device in the network and it will work. It doesn’t have to be on a local network or on the same Wifi.

We can also change the format of our IP by selecting a different IP format from ipv4 auto-assign (refer image 2 of step 4 in Creating a Zerotier Network. The new IPs will be added. Note that the old ones will not be deleted, we have to delete them manually.

We can also manually add more IPs for a device by writing them in the text box under their specific IP in the network dashboard and clicking the ‘+’ button. But the IP should be in the exact form as the one in the selected IP auto assign format. We can only change the part in ‘*’.

What’s next

So, that’s how we create and add a device to a Zerotier network. After everything is configured we can use that IP for sshing into a computer on the network or playing multiplayer games on the internet without having an actual public IP. Note that this method works as long as the clients are connected to the same network.
